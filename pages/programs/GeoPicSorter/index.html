GeoPicSorter is a free and open source program written in Java that will automate the process of organizing pictures based on groups of geolocations such as streets, cities, etc. It's in its early stages of development, contributions of any kind are greatly appreciated!

Which tools were used?

GeoPicSorter's development was possible thanks to the following third-party tools:

  • Metadata-extractor: extract metadata from pictures, used to get the latitude and longitude from pictures.
  • SQLite JDBC: to manage SQLite databases, used to store configurations and data relevant to the picture sorting process like file location and coordinates.
  • OkHttp: used to send HTTP requests to the online tool I use for translating coordinates into real life location data.
  • Geoapify: they offer a free reverse geocoding API that we can use to get that data I've just talked about. By default, this service is used and a free API key is supplied for test usage right out the box.
  • JSON-java: because the data is returned in JSON format, it's necessary to selectively extract it, it's first received in a string then parsed as JSON, where the required info can be extracted, like the street name.
What do I need to run it?
  • 16MB of disk storage.
  • Latest Java SE installation*.
  • Internet connection.

*Java SE is required because of support for class version 62 files, which the Java 8 installation doesn't come with (it goes up to version 52). You can download Java SE from the official website at Oracle.

How do I download it?

Simply download the latest release from its repository, it's a JAR file that can be opened via a double click or by using the java -jar command.
